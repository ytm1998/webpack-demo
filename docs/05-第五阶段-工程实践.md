## 任务 5.1：脚手架集成与问题排查

### 5.1.1 核心概念

#### 主流脚手架对比

| 脚手架 | 构建工具 | 特点 | 适用场景 |
|--------|----------|------|----------|
| Create React App | Webpack 4→5 | 开箱即用 | React 项目快速启动 |
| Vue CLI | Webpack | Vue 官方 | Vue 2 项目 |
| Vite | 原生 ESM + Go | 极速开发 | Vue 3 / 现代项目 |
| Next.js | Webpack | SSR/SSG | React 全栈框架 |
| Umi | Webpack | 企业级 | 中后台应用 |

#### Create React App (CRA)

**结构**：
```
my-app/
├── config/           # Webpack 配置（隐藏）
├── public/
├── scripts/         # 构建脚本
├── src/
│   ├── App.js
│   └── index.js
├── package.json
└── node_modules/
```

**问题**：配置隐藏，难以自定义

**解决方案**：使用 `react-scripts eject`（不可逆）或 `react-app-rewired`

### 5.1.2 Webpack 5 迁移指南

#### 主要变更

| 变更 | Webpack 4 | Webpack 5 |
|------|-----------|-----------|
| 输出清理 | clean-webpack-plugin | output.clean: true |
| 文件资源 | file-loader/url-loader | asset/resource, asset/inline |
| Node Polyfills | 自动引入 | 需要手动配置 |
| 缓存 | 内存 | 文件系统 |
| 输出文件名 | hash/chunkhash | contenthash |

#### 迁移步骤

**1. 更新依赖**

```bash
# 卸载旧版本
npm uninstall webpack webpack-cli webpack-dev-server

# 安装 Webpack 5
npm install webpack@5 webpack-cli webpack-dev-server --save-dev
```

**2. 更新 Loader 配置**

```javascript
// Webpack 4
{
  test: /\.png$/,
  use: [
    {
      loader: 'file-loader',
      options: {
        name: 'images/[name].[hash:8].[ext]',
      },
    },
  ],
}

// Webpack 5
{
  test: /\.png$/,
  type: 'asset/resource',
  generator: {
    filename: 'images/[name].[hash:8][ext]',
  },
}
```

**3. 更新输出配置**

```javascript
// Webpack 4
const CleanWebpackPlugin = require('clean-webpack-plugin');

module.exports = {
  plugins: [
    new CleanWebpackPlugin(['dist']),
  ],
  output: {
    filename: '[name].[chunkhash].js',
  },
};

// Webpack 5
module.exports = {
  output: {
    filename: '[name].[contenthash].js',
    clean: true,  // 自动清理
  },
};
```

**4. 处理 Node Polyfills**

```javascript
// Webpack 5 不再自动引入 polyfills
module.exports = {
  resolve: {
    fallback: {
      path: false,   // 不需要 path polyfill
      fs: false,    // 不需要 fs polyfill
    },
  },
  externals: {
    // 某些库需要作为 external
  },
};
```

### 5.1.3 常见问题排查

#### 问题 1：内存溢出（Out of Memory）

**症状**：
```
FATAL ERROR: CALL_AND_RETRY_LAST Allocation failed - JavaScript heap out of memory
```

**解决方案**：

```bash
# 增加 Node 内存限制
NODE_OPTIONS=--max_old_space_size=4096 npm run build
```

```json
// package.json
{
  "scripts": {
    "build": "NODE_OPTIONS=--max_old_space_size=4096 webpack --mode production"
  }
}
```

**长期解决方案**：
- 使用 esbuild-loader 替换 babel-loader
- 开启持久化缓存
- 减少并行构建进程数
- 分批构建

---

#### 问题 2：模块找不到

**症状**：
```
ERROR in Cannot resolve module 'xxx'
```

**排查步骤**：

1. **检查路径**
```javascript
resolve: {
  extensions: ['.js', '.jsx', '.json'],
  alias: {
    '@': path.resolve(__dirname, 'src'),
  },
}
```

2. **检查大小写**
```javascript
// Linux/Mac 文件系统区分大小写
// 确保 import 路径与实际文件名大小写一致
import MyComponent from './MyComponent';  // ✓
import MyComponent from './myComponent';  // ✗
```

3. **检查 node_modules**
```javascript
resolve: {
  modules: [
    path.resolve(__dirname, 'src'),
    'node_modules',
  ],
}
```

---

#### 问题 3：构建速度太慢

**排查工具**：speed-measure-webpack-plugin

```javascript
const SpeedMeasurePlugin = require('speed-measure-webpack-plugin');
const smp = new SpeedMeasurePlugin();

module.exports = smp.wrap({
  // 你的配置
});
```

**常见原因**：

| 原因 | 解决方案 |
|------|----------|
| babel-loader 太慢 | 使用 esbuild-loader |
| 缺少缓存 | 开启 filesystem 缓存 |
| 太多 Loader | 使用 thread-loader 并行 |
| 重复解析 | 配置 resolve.modules |

---

#### 问题 4：CSS 不生效

**症状**：修改 CSS 后页面没有变化

**常见原因**：

1. **缓存问题**
```javascript
// 清除缓存
rm -rf node_modules/.cache
```

2. **Loader 顺序错误**
```javascript
// 正确顺序：先解析，再插入 DOM
{
  test: /\.css$/,
  use: ['style-loader', 'css-loader'],
}
```

3. **MiniCssExtractPlugin 冲突**
```javascript
// 开发环境使用 style-loader
// 生产环境使用 MiniCssExtractPlugin.loader
const isProd = process.env.NODE_ENV === 'production';

{
  test: /\.css$/,
  use: [
    isProd ? MiniCssExtractPlugin.loader : 'style-loader',
    'css-loader',
  ],
}
```

---

#### 问题 5：热更新不生效

**症状**：修改代码后页面刷新但内容不变

**解决方案**：

1. **检查 HMR 配置**
```javascript
module.exports = {
  devServer: {
    hot: true,
    liveReload: false,
  },
  plugins: [
    new webpack.HotModuleReplacementPlugin(),
  ],
};
```

2. **检查代码**
```javascript
// 确保有 module.hot 判断
if (module.hot) {
  module.hot.accept();
}
```

3. **检查异步加载**
```javascript
// 异步加载的模块 HMR 需要特殊处理
import(/* webpackChunkName: "detail" */ './Detail').then(module => {
  const Detail = module.default;
  // ...
});

// HMR
if (module.hot) {
  module.hot.accept();
}
```

### 5.1.4 Vite 对比与选型

#### Vite 特点

| 特性 | Webpack | Vite |
|------|---------|------|
| 开发启动 | 慢（打包） | 快（原生 ESM） |
| HMR | 基于字节 | 基于 ESM |
| 生产构建 | Webpack | Rollup |
| 生态 | 成熟 | 发展中 |

#### Vite 原理

```
开发环境：
┌────────────────────────────────────┐
│           Vite Dev Server          │
├────────────────────────────────────┤
│  1. 浏览器请求 index.js            │
│  2. Vite 拦截请求                  │
│  3. 按需编译（仅当前文件）         │
│  4. 返回编译后的模块                │
└────────────────────────────────────┘

vs

Webpack：
┌────────────────────────────────────┐
│          Webpack Dev Server        │
├────────────────────────────────────┤
│  1. 启动时全量打包                 │
│  2. 浏览器请求                     │
│  3. 返回完整的 bundle              │
└────────────────────────────────────┘
```

#### 何时选择 Vite

| 场景 | 推荐 |
|------|------|
| 新项目 | ✅ Vite |
| Vue 3 项目 | ✅ Vite |
| 需要 SSR | Next.js / Nuxt |
| 大型企业项目 | Webpack |
| 需要兼容旧浏览器 | Webpack |

## 任务 5.2：企业级配置模板

### 5.2.1 完整配置模板

#### 目录结构

```
project/
├── config/
│   ├── webpack.common.js
│   ├── webpack.dev.js
│   └── webpack.prod.js
├── src/
├── public/
├── .babelrc.js
├── .browserslistrc
├── package.json
└── webpack.config.js
```

#### webpack.common.js

```javascript
const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');
const { VueLoaderPlugin } = require('vue-loader');

const resolve = dir => path.resolve(__dirname, '..', dir);

module.exports = {
  entry: './src/index.js',
  output: {
    path: resolve('dist'),
    filename: '[name].[contenthash:8].js',
    chunkFilename: '[name].[contenthash:8].chunk.js',
    clean: true,
    publicPath: '/',
  },
  resolve: {
    extensions: ['.js', '.jsx', '.ts', '.tsx', '.vue', '.json'],
    alias: {
      '@': resolve('src'),
      'vue': 'vue/dist/vue.esm-bundler.js',
    },
  },
  module: {
    rules: [
      // Vue
      {
        test: /\.vue$/,
        loader: 'vue-loader',
      },
      // JS/TS
      {
        test: /\.(js|jsx|ts|tsx)$/,
        exclude: /node_modules/,
        use: {
          loader: 'babel-loader',
          options: {
            cacheDirectory: true,
          },
        },
      },
      // CSS
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader'],
      },
      // 图片
      {
        test: /\.(png|jpe?g|gif|svg)$/i,
        type: 'asset/resource',
        generator: {
          filename: 'images/[name].[hash:8][ext]',
        },
      },
      // 字体
      {
        test: /\.(woff2?|eot|ttf|otf)$/i,
        type: 'asset/resource',
        generator: {
          filename: 'fonts/[name].[hash:8][ext]',
        },
      },
    ],
  },
  plugins: [
    new VueLoaderPlugin(),
    new HtmlWebpackPlugin({
      template: './public/index.html',
      inject: true,
    }),
  ],
  optimization: {
    splitChunks: {
      chunks: 'all',
      cacheGroups: {
        vendors: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          priority: 10,
        },
        common: {
          minChunks: 2,
          priority: 5,
          reuseExistingChunk: true,
        },
      },
    },
    runtimeChunk: 'single',
  },
};
```

#### webpack.dev.js

```javascript
const { merge } = require('webpack-merge');
const common = require('./webpack.common');

module.exports = merge(common, {
```

#### webpack.prod.js

```javascript
const { merge } = require('webpack-merge');
const common = require('./webpack.common');
const TerserPlugin = require('terser-webpack-plugin');
const CssMinimizerPlugin = require('css-minimizer-webpack-plugin');
const CompressionPlugin = require('compression-webpack-plugin');

module.exports = merge(common, {
  mode: 'production',
  devtool: 'source-map',
  output: {
    filename: '[name].[contenthash:8].js',
    chunkFilename: '[name].[contenthash:8].chunk.js',
  },
  optimization: {
    minimize: true,
    minimizer: [
      new TerserPlugin({
        terserOptions: {
          compress: {
            drop_console: true,
            drop_debugger: true,
          },
        },
      }),
      new CssMinimizerPlugin(),
    ],
  },
  plugins: [
    new CompressionPlugin({
      algorithm: 'gzip',
      test: /\.(js|css|html|svg)$/,
      threshold: 10240,
    }),
  ],
});
```

### 5.2.2 性能优化配置

#### 完整优化配置

```javascript
module.exports = {
  // 缓存
  cache: {
    type: 'filesystem',
    buildDependencies: {
      config: [__filename],
    },
  },
  
  // 优化
  optimization: {
    // 模块 ID 稳定

    chunkIds: 'deterministic',
    
    // Tree Shaking

    sideEffects: true,
    
    // 代码分割
    splitChunks: {
      chunks: 'all',
      maxInitialRequests: 25,
      maxAsyncRequests: 25,
      minSize: 20000,
      cacheGroups: {
        vendors: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',
          priority: 10,
        },
      },
    },
    
    // 运行时
    runtimeChunk: 'single',
  },
  
  // 性能
  performance: {
    hints: 'warning',
    maxEntrypointSize: 512000,
    maxAssetSize: 512000,
  },
};
```
