## 任务 3.1：构建速度优化

### 3.1.1 核心概念

#### 为什么构建慢
- 大量文件需要处理
- 重复解析相同的模块
- 缺少缓存
- 单线程执行

#### 优化策略
| 策略 | 效果 | 适用场景 |
|------|------|----------|
| 持久化缓存 | ⭐⭐⭐⭐⭐ | 所有项目 |
| 并行构建 | ⭐⭐⭐⭐ | 大型项目 |
| esbuild/swc | ⭐⭐⭐⭐⭐ | 新项目/追求速度 |
| 外部扩展 | ⭐⭐⭐ | 有 CDN 的项目 |
| Source Map 优化 | ⭐⭐⭐ | 开发环境 |

---

### 3.1.2 缓存机制

#### Webpack 5 持久化缓存

```javascript
module.exports = {
  cache: {
    type: 'filesystem',      // 持久化到磁盘
    buildDependencies: {    // 依赖变化时清除缓存
      config: [__filename],
    },
    compression: 'gzip',    // 压缩缓存文件（减小磁盘占用）
    cacheDirectory: path.resolve(__dirname, '.webpack-cache'),
  },
};
```

**缓存效果对比**：
```
首次构建：3000ms
二次构建：300ms  （90% 提升）
```

#### Babel Loader 缓存

```javascript
{
  test: /\.js$/,
  exclude: /node_modules/,
  use: {
    loader: 'babel-loader',
    options: {
      cacheDirectory: true,      // 启用缓存
      cacheCompression: false,   // 不压缩（更快）
    },
  },
}
```

### 3.1.4 Source Map 类型对比

| 类型 | 构建速度 | 重建速度 | 适用场景 |
|------|----------|----------|----------|
| `eval` | 最快 | 最快 | 大型项目开发 |
| `cheap-eval-source-map` | 快 | 快 | 一般开发 |
| `eval-source-map` | 较慢 | 快 | 首次开发 |
| `cheap-module-eval-source-map` | 中等 | 快 | 推荐开发 |
| `inline-source-map` | 慢 | 较快 | 小项目 |
| `source-map` | 最慢 | 最慢 | 生产环境 |

```javascript
// 开发环境推荐
module.exports = {
  mode: 'development',
  devtool: 'eval-cheap-module-source-map',
};

// 生产环境
module.exports = {
  mode: 'production',
  devtool: 'source-map',  // 不会影响运行性能
};
```

### 3.2.1 核心概念

#### 产物优化目标

| 策略 | 效果 |
|------|------|
| Tree Shaking | 删除未使用代码 |

| 压缩 | Terser、CssMinimizer |

---

### 3.2.2 Tree Shaking

#### 原理
基于 ES Module 的**静态分析**，在编译时确定模块删除未使用的导出。

#### 开启条件
依赖关系，1. 使用 ES Module 语法（import/export）
2. production 模式（或开启 usedExports）

```javascript
module.exports = {
  mode: 'production',  // 自动开启
  
  // 或手动配置
  optimization: {
    usedExports: true,
    sideEffects: true,
  },
};
```

#### sideEffects 配置

```json
// package.json
{
  "name": "my-lib",
  "sideEffects": [
    "**/*.css",           // CSS 有副作用，不能删除
    "**/*.less",
    "./es/index.js"      // 特定文件有副作用
  ]
}
```

```javascript
// webpack.config.js
module.exports = {
  optimization: {
    sideEffects: true,   // 读取 package.json
  },
};
```

#### Tree Shaking 示例

```javascript
// math.js
export function add(a, b) {
  return a + b;
}

export function multiply(a, b) {
  return a * b;
}

export function subtract(a, b) {
  return a - b;
}
```

```javascript
// index.js
import { add } from './math';

console.log(add(1, 2));
```

**打包结果**：只包含 `add` 函数，`multiply` 和 `subtract` 被删除

### 3.2.3 Code Splitting（代码分割）

#### 基本配置

```javascript
module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all',           // 分割同步+异步导入
      minSize: 20000,          // 最小 chunk 大小（字节）
      maxSize: 244000,         // 最大 chunk 大小
      minChunks: 1,            // 最小被引用次数
      maxAsyncRequests: 30,   // 最大异步请求数
      maxInitialRequests: 30, // 最大初始请求数
    },
  },
};
```

#### 缓存组配置

```javascript
splitChunks: {
  chunks: 'all',
  cacheGroups: {
    // 1. 提取 node_modules
    vendors: {
      test: /[\\/]node_modules[\\/]/,
      name: 'vendors',
      priority: -10,
      reuseExistingChunk: true,
    },
    
    // 2. 提取 React 相关库
    react: {
      test: /[\\/]node_modules[\\/](react|react-dom|react-router)[\\/]/,
      name: 'react',
      chunks: 'all',
      priority: 20,
    },
    
    // 3. 提取公共代码
    common: {
      minChunks: 2,
      priority: -20,
      reuseExistingChunk: true,
    },
  },
},

// 4. 运行时提取
optimization: {
  runtimeChunk: 'single',
}
```

#### splitChunks 完整配置

```javascript
splitChunks: {
  chunks: 'all',
  minSize: 20000,
  maxSize: 0,
  minChunks: 1,
  maxAsyncRequests: 30,
  maxInitialRequests: 30,
  cacheGroups: {
    defaultVendors: {
      test: /[\\/]node_modules[\\/]/,
      priority: -10,
      reuseExistingChunk: true,
      name: 'vendors',
    },
    default: {
      minChunks: 2,
      priority: -20,
      reuseExistingChunk: true,
    },
  },
}
```

### 3.2.4 动态导入（懒加载）

#### 基础用法

```javascript
// 路由懒加载 - 访问时加载
```

#### React Router 示例

```javascript
import { Suspense, lazy } from 'react';
import { Routes, Route } from 'react-router-dom';

const Home = lazy(() => import('./pages/Home'));
const About = lazy(() => import('./pages/About'));

function App() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </Suspense>
  );
}
```

#### Vue Router 示例

```javascript
const Home = () => import(/* webpackChunkName: "home" */ './pages/Home');
const About = () => import(/* webpackChunkName: "about" */ './pages/About');

const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About },
];
```

### 3.2.5 预加载/预获取


| 方式 | 加载时机 | 使用场景 |
|------|----------|----------|

| preload | 立即加载 | 当前页面需要但不是关键 |

### 3.3.2 缓存策略

#### hash vs contenthash

```javascript
output: {
  // [name] - 入口名称
  // [id] - 模块 ID
  // [chunkhash] - chunk 内容的 hash
  // [contenthash] - 文件内容的 hash（推荐）
  
  filename: '[name].[contenthash:8].js',
  chunkFilename: '[name].[contenthash:8].chunk.js',
}
```

#### 稳定模块 ID

```javascript
module.exports = {
  optimization: {
    moduleIds: 'deterministic',  // 模块 ID 稳定
    chunkIds: 'deterministic',   // chunk ID 稳定
  },
};
```

**对比**：
| 模式 | 说明 |
|------|------|
| `natural` | 按引用顺序（默认） |
| `named` | 可读性好，开发用 |
| `deterministic` | 稳定，生产用 |

### 3.3.3 模块联邦（Module Federation）

#### 概念
Webpack 5 的核心特性，允许跨应用共享模块，是微前端的基础。

#### 核心概念
| 概念 | 说明 |
|------|------|
| Host | 消费方，使用远程模块 |
| Remote | 提供方，暴露本地模块 |
| Shared | 共享依赖 |

#### 基础配置

**应用 A（提供方）**
```javascript
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');

module.exports = {
  plugins: [
    new ModuleFederationPlugin({
      name: 'app1',
      filename: 'remoteEntry.js',
      exposes: {
        './Button': './src/Button',
        './Header': './src/Header',
      },
      shared: {
        react: { singleton: true, eager: true },
        'react-dom': { singleton: true, eager: true },
      },
    }),
  ],
};
```

**应用 B（消费方）**
```javascript
module.exports = {
  plugins: [
    new ModuleFederationPlugin({
      name: 'app2',
      remotes: {
        app1: 'app1@http://localhost:3001/remoteEntry.js',
      },
      shared: {
        react: { singleton: true },
        'react-dom': { singleton: true },
      },
    }),
  ],
};
```

**使用远程模块**
```javascript
import React from 'react';
import { Suspense } from 'react';

const Button = React.lazy(() => import('app1/Button'));

function App() {
  return (
    <Suspense fallback="Loading...">
      <Button onClick={() => alert('Clicked!')}>
        Click Me
      </Button>
    </Suspense>
  );
}
```

#### Shared 配置详解

```javascript
shared: {
  react: {
    singleton: true,       // 单例模式（推荐）
    eager: true,           // 预加载
    requiredVersion: '^18.0.0',  // 版本要求
    shareKey: 'react',     // 共享键名
    shareScope: 'default', // 共享域
  },
}
```

#### 实战 2：搭建 Module Federation 微前端

**目录结构**：
```
micro-frontend/
├── host-app/      # 主应用
│   ├── src/
│   ├── webpack.config.js
│   └── package.json
└── remote-app/   # 远程应用
    ├── src/
    ├── webpack.config.js
    └── package.json
```

**remote-app 配置**：
```javascript
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');
const HtmlWebpackPlugin = require('html-webpack-plugin');
const path = require('path');

module.exports = {
  entry: './src/index',
  mode: 'development',
  devServer: {
    port: 3001,
  },
  plugins: [
    new ModuleFederationPlugin({
      name: 'remote_app',
      filename: 'remoteEntry.js',
      exposes: {
        './Button': './src/Button',
      },
      shared: {
        react: { singleton: true, eager: true },
        'react-dom': { singleton: true, eager: true },
      },
    }),
    new HtmlWebpackPlugin({
      template: './public/index.html',
    }),
  ],
  resolve: {
    extensions: ['.jsx', '.js'],
  },
};
```

**host-app 配置**：
```javascript
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');
const HtmlWebpackPlugin = require('html-webpack-plugin');
const path = require('path');

module.exports = {
  entry: './src/index',
  mode: 'development',
  devServer: {
    port: 3000,
  },
  plugins: [
    new ModuleFederationPlugin({
      name: 'host_app',
      remotes: {
        remote_app: 'remote_app@http://localhost:3001/remoteEntry.js',
      },
      shared: {
        react: { singleton: true, eager: true },
        'react-dom': { singleton: true, eager: true },
      },
    }),
    new HtmlWebpackPlugin({
      template: './public/index.html',
    }),
  ],
  resolve: {
    extensions: ['.jsx', '.js'],
  },
  module: {
    rules: [
      {
        test: /\.jsx?$/,
        loader: 'babel-loader',
        exclude: /node_modules/,
        options: {
          presets: ['@babel/preset-react'],
        },
      },
    ],
  },
};
```
