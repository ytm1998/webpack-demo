## 任务 2.1：Loader 机制

### 2.1.1 核心概念

#### 什么是 Loader
- **Loader 本质**：函数（`function(source, sourcemap, meta)`）
- **作用**：处理非 JavaScript 文件（CSS、图片、TypeScript、Vue 等）
- **返回**：转换后的 JavaScript 代码

#### Loader 的分类

| 类型 | 说明 | 示例 |
|------|------|------|
| **同步 Loader** | 同步返回结果 | `return` |
| **异步 Loader** | 异步返回结果 | `this.async()` |
| **Pitch Loader** | 先于普通 Loader 执行 | `pitch` 函数 |

#### Loader 链式调用
- **从右到左**执行：`use: ['a', 'b', 'c']` → 先执行 c，最后执行 a
- **从下到上**执行：`rule.use` 数组从后往前
- 可以通过 `pitch` 阶段改变执行顺序

```javascript
// 链式调用示例
module: {
  rules: [
    {
      test: /\.less$/,
      use: [
        'style-loader',   // 1. 插入到 DOM（最后执行）
        'css-loader',     // 2. 解析 CSS 中的 @import 和 url()
        'less-loader',    // 3. 编译 LESS 为 CSS（最先执行）
      ],
    },
  ],
}
```

#### Loader 上下文（this）

```javascript
// this 包含丰富的 API
module.exports = function(source) {
  // this.query - 获取 Loader 配置（已废弃，推荐 getOptions）
  const options = this.getOptions();
  
  // this.emitFile - 发射额外文件
  this.emitFile('sprite.png', buffer);
  
  // this.addDependency - 添加依赖
  this.addDependency(filePath);
  
  // this.getOptions - 获取配置
  // this.callback - 返回多个结果
  // this.async - 异步执行
  
  return source;
};
```

## 任务 2.2：Plugin 机制

### 2.2.1 核心概念

#### 什么是 Plugin
- **Plugin 本质**：类（Class），需要实现 `apply(compiler)` 方法
- **作用**：扩展 Webpack 功能，处理各种构建任务
- **生命周期**：通过钩子函数监听 Webpack 事件

#### Compiler vs Compilation

| 对象 | 说明 | 生命周期 |
|------|------|----------|
| **Compiler** | 全局编译对象，整个构建过程只创建一次 | run → compile → watch |
| **Compilation** | 单次编译实例，每次构建都会创建 | compile → emit |

#### Tapable 事件流

Webpack 基于 Tapable 的发布订阅模式：

```javascript
// Tapable 钩子类型
compiler.hooks.emit.tap('PluginName', (compilation) => {
  // 处理逻辑
});

// 常用钩子类型
- tap          - 同步钩子
- tapAsync     - 异步钩子（回调）
- tapPromise   - 异步钩子（Promise）
```

#### 常用钩子列表

| 钩子 | 阶段 | 说明 |
|------|------|------|
| `entryOption` | 初始化后 | 处理入口配置 |
| `afterPlugins` | 设置插件后 | 插件注册完成 |
| `compile` | 编译开始前 | 开始编译 |
| `compilation` | 编译中 | 创建 Compilation |
| `optimize` | 优化阶段 | 开始优化 |
| `emit` | 资源发射前 | 生成资源到目录前（常用） |
| `afterEmit` | 资源发射后 | 资源已生成 |
| `done` | 完成后 | 构建完成 |

### 2.2.2 手写 Plugin

#### 基础结构

```javascript
class MyPlugin {
  constructor(options) {
    this.options = options || {};
  }
  
  apply(compiler) {
    // 监听钩子
    compiler.hooks.emit.tap('MyPlugin', (compilation) => {
      console.log('编译中...');
    });
  }
}

module.exports = MyPlugin;
```

#### 完整示例：添加版权注释

```javascript
class BannerPlugin {
  constructor(options) {
    this.options = {
      banner: 'Generated by Webpack',
      ...options,
    };
  }
  
  apply(compiler) {
    // 监听 emit 钩子（在生成资源之前）
    compiler.hooks.emit.tap('BannerPlugin', (compilation) => {
      // 遍历所有资源
      for (const [filename, asset] of Object.entries(compilation.assets)) {
        if (filename.endsWith('.js')) {
          const source = asset.source();
          const content = `/* ${this.options.banner} */\n${source}`;
          
          // 更新资源
          compilation.assets[filename] = {
            source: () => content,
            size: () => content.length,
          };
        }
      }
    });
  }
}
```

#### 异步 Plugin 示例

```javascript
class AsyncPlugin {
  apply(compiler) {
    // 异步钩子使用 tapAsync 或 tapPromise
    compiler.hooks.emit.tapAsync('AsyncPlugin', (compilation, callback) => {
      // 异步操作
      setTimeout(() => {
        console.log('Async operation done');
        callback();  // 必须调用 callback
      }, 100);
    });
    
    // 或者使用 tapPromise
    compiler.hooks.emit.tapPromise('AsyncPlugin', (compilation) => {
      return doAsyncOperation().then(() => {
        console.log('Promise resolved');
      });
    });
  }
}
```

#### 实战 2：手写 RemoveConsolePlugin

**目标**：生产环境移除 console.log

```javascript
class RemoveConsolePlugin {
  constructor(options) {
    this.options = {
      exclude: ['error', 'warn'],  // 保留的 console 类型
      ...options,
    };
  }
  
  apply(compiler) {
    compiler.hooks.compilation.tap('RemoveConsolePlugin', (compilation) => {
      compilation.hooks.optimizeChunks.tap('RemoveConsolePlugin', (chunks) => {
        chunks.forEach(chunk => {
          chunk.modules.forEach(module => {
            if (module._source) {
              const source = module._source._value;
              
              // 移除 console.log 等
              const excludeSet = new Set(this.options.exclude);
              
              const newSource = source
                .replace(/console\.(log|info|debug)\([^)]*\);?/g, '')
                .replace(/console\.(log|info|debug)`[^`]*`;?/g, '')
                .replace(/debugger;?/g, '');
              
              module._source._value = newSource;
            }
          });
        });
      });
    });
  }
}

module.exports = RemoveConsolePlugin;
```

**使用配置**

#### 实战 3：手写 AnalyzePlugin

**目标**：统计各模块大小

```javascript
class AnalyzePlugin {
  apply(compiler) {
    compiler.hooks.done.tap('AnalyzePlugin', (stats) => {
      const info = stats.toJson({
        chunks: true,
        chunkModules: true,
      });
      
      const moduleSizes = {};
      
      info.chunks.forEach(chunk => {
        chunk.modules.forEach(module => {
          const size = module.size || 0;
          const name = module.name || module.identifier;
          
          if (!moduleSizes[name]) {
            moduleSizes[name] = 0;
          }
          moduleSizes[name] += size;
        });
      });
      
      // 按大小排序
      const sorted = Object.entries(moduleSizes)
        .sort((a, b) => b[1] - a[1])
        .slice(0, 20);
      
      console.log('\n=== Module Size Analysis ===');
      sorted.forEach(([name, size]) => {
        console.log(`${(size / 1024).toFixed(2)} KB - ${name}`);
      });
    });
  }
}

module.exports = AnalyzePlugin;
```

#### 依赖解析流程

```
1. 读取入口文件
2. 解析文件路径（相对路径/绝对路径/模块名）
3. 查找文件（resolve 配置）
4. 读取文件内容
5. 分析依赖（AST 解析）
6. 递归处理依赖
7. 构建模块依赖图
```

#### 实战 3：理解动态导入

// 静态导入 - 同步
import { add } from './math';

// 动态导入 - 异步（懒加载）
const mathModule = import('./math');
mathModule.then(math => {
  console.log(math.add(1, 2));
});
