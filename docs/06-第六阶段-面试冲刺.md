## 一、高频面试题汇总

### 1.1 基础概念题

#### Q1: Webpack 和 Gulp/Grunt 的区别？

| 工具 | 类型 | 核心思想 | 适用场景 |
|------|------|----------|----------|
| **Gulp/Grunt** | 任务流工具 | 自动化任务（压缩、合并） | 简单项目 |
| **Webpack** | 模块打包器 | 模块依赖打包 | 现代模块化项目 |

**参考答案**：
- Gulp/Grunt 是基于任务的自动化工具，通过配置一系列任务来处理文件
- Webpack 是模块化打包工具，核心是解决模块依赖问题
- Webpack 天然支持 Code Splitting、Tree Shaking 等优化
- 对于复杂的前端项目（React/Vue），Webpack 更适合

---

#### Q2: Webpack 5 有哪些新特性？

**参考答案**：
1. **持久化缓存** - `cache: { type: 'filesystem' }`，大幅提升二次构建速度
2. **模块联邦（Module Federation）** - 跨应用共享模块，微前端基础
3. **Asset 模块** - 替代 file-loader/url-loader
4. **Tree Shaking 改进** - 更好的静态分析
5. **Output 改进** - `output.clean: true` 自动清理
6. **移除 Node Polyfills** - 不再自动引入，更灵活

---

#### Q3: Webpack 的构建流程？

**参考答案**：
1. **初始化** - 读取配置，初始化 Compiler
2. **编译** - 从入口文件开始，递归解析依赖
3. **模块处理** - 调用 Loader 处理文件，分析 AST
4. **代码生成** - 生成 Chunk，构建 Bundle
5. **输出** - 写入文件系统

### 1.2 Loader/Plugin 题

#### Q4: Loader 和 Plugin 的区别？

| 对比 | Loader | Plugin |
|------|--------|--------|
| **本质** | 函数 | 类 |
| **作用** | 文件转换 | 扩展功能 |
| **执行时机** | 编译时 | 整个生命周期 |
| **配置方式** | module.rules | plugins 数组 |
| **处理方式** | 一对一 | 一对多 |

**参考答案**：
- Loader 本质是一个函数，用于将非 JS 文件转换为 JS 模块
- Plugin 是类，通过监听 Webpack 生命周期钩子来扩展功能
- Loader 从右到左、从下到上链式执行
- Plugin 可以监听多个钩子，处理多种任务

### 1.3 性能优化题（重点）

#### Q7: Tree Shaking 原理？

**参考答案**：
- 基于 ES Module 的**静态分析**
- 在编译时确定模块的导出和导入关系
- 删除未使用的导出（used exports）
- 条件：
  1. 使用 ES Module 语法（import/export）
  2. production 模式
  3. 配置 `sideEffects`

```javascript
// math.js
export function add() {}
export function multiply() {}  // 未使用，会被删除

// index.js
import { add } from './math';
add(1, 2);  // multiply 会被 Tree Shaking 删除
```

#### Q8: 如何做 Code Splitting？

**参考答案**：

**方式1：配置 splitChunks**
```javascript
optimization: {
  splitChunks: {
    chunks: 'all',
    cacheGroups: {
      vendors: {
        test: /[\\/]node_modules[\\/]/,
        name: 'vendors',
        priority: 10,
      },
    },
  },
}
```

**方式2：动态导入**
```javascript
// 路由懒加载
const Home = () => import('./pages/Home');

// React
const Modal = React.lazy(() => import('./Modal'));
```

### 1.4 原理题

#### Q10: Webpack 热更新原理？

**参考答案**：
1. 监听文件变化
2. Webpack 重新编译
3. 通过 WebSocket 推送消息给浏览器
4. 浏览器加载新的模块代码
5. 调用 `module.hot.accept` 替换模块
6. 保留页面状态

```
文件变化 → 重新编译 → WebSocket 推送 → 替换模块 → 状态保留
```

## 三、手写代码题

### 3.1 手写简易版 Webpack

```javascript
const fs = require('fs');
const path = require('path');

class SimpleWebpack {
  constructor(options) {
    this.options = options;
    this.modules = {};
  }
  
  run() {
    const entryId = this.buildModule(this.options.entry);
    this.emitBundle(entryId);
  }
  
  buildModule(modulePath) {
    const code = fs.readFileSync(modulePath, 'utf-8');
    const id = './' + path.relative(process.cwd(), modulePath);
    
    // 简化：只处理 import
    const deps = [];
    code.replace(/import .* from ['"](.+)['"]/g, (_, dep) => {
      deps.push(dep);
    });
    
    this.modules[id] = code;
    
    deps.forEach(dep => {
      const depPath = path.resolve(path.dirname(modulePath), dep);
      this.buildModule(depPath);
    });
    
    return id;
  }
  
  emitBundle(entryId) {
    const bundle = `
(function(modules) {
  function require(id) {
    const fn = modules[id];
    const module = { exports: {} };
    fn(module, module.exports, require);
    return module.exports;
  }
  require('${entryId}');
})(${JSON.stringify(this.modules)})
    `.trim();
    
    fs.writeFileSync(
      path.join(this.options.output.path, this.options.output.filename),
      bundle
    );
  }
}

module.exports = SimpleWebpack;
```

### 3.3 手写 Plugin

```javascript
class BannerPlugin {
  constructor(options) {
    this.options = options;
  }
  
  apply(compiler) {
    compiler.hooks.emit.tap('BannerPlugin', (compilation) => {
      for (const [name, asset] of Object.entries(compilation.assets)) {
        if (name.endsWith('.js')) {
          const source = asset.source();
          const banner = `// ${this.options.banner}\n`;
          compilation.assets[name] = {
            source: () => banner + source,
            size: () => source.length + banner.length,
          };
        }
      }
    });
  }
}
```
